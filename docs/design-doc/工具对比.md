# Spec Kit Code Review Guide

本文档面向第一次使用 Spec Kit 做代码审查的用户，目标是用最少的判断成本，完成以下事情：

1. 安装 VS Code Copilot、Copilot CLI、Claude Code
2. 在项目里初始化 Spec Kit
3. 使用 `/speckit.code-review` 发起代码审查
4. 看懂审查结果落在哪里，出问题时怎么排查

> [!NOTE]
> 在 Spec Kit 当前实现里，`Copilot CLI` 和 `VS Code Copilot` 共用同一个 `--ai copilot` 适配层。
> 也就是说，这两个工具的 Spec Kit 初始化命令相同，主要区别在运行环境不同：一个在终端里用，一个在 VS Code 里用。

## 1. 先看结论

如果你只想最快跑起来，按下面选：

| 你要用的工具 | 适合谁 | `specify init` 参数 | 初始化后重点检查的目录 |
| --- | --- | --- | --- |
| VS Code Copilot | 喜欢在 VS Code Chat 里操作的人 | `--ai copilot` | `.github/agents/`、`.github/prompts/` |
| Copilot CLI | 喜欢在终端里操作的人 | `--ai copilot` | `.github/agents/` |
| Claude Code | 喜欢纯 CLI 工作流的人 | `--ai claude` | `.claude/commands/` |

审查意图是统一的，但不同宿主的触发方式不完全一样：

| 运行环境 | 你实际输入的方式 |
| --- | --- |
| VS Code Copilot | 直接输入 `/speckit.code-review review ...` |
| Copilot CLI | 先选 agent `speckit.code-review`，再输入 `review ...` |
| Claude Code | 直接输入 `/speckit.code-review review ...` |

其中，统一的审查请求内容可以写成：

```text
review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

把上面的 `张三`、`zhangsan`、`op-api`、`2026-04-13` 换成你的真实信息即可。

## 2. 通用前置条件

先装好这几样基础工具：

1. Git
2. Node.js 和 npm
3. `uv` 或 `uvx`
4. Spec Kit CLI

### 2.1 Windows 推荐安装命令

如果你已经装过，可以跳过。

```powershell
winget install Git.Git
winget install OpenJS.NodeJS.LTS
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

安装完成后，重新打开 PowerShell，执行：

```powershell
git --version
node --version
npm --version
uv --version
```

### 2.2 安装 Spec Kit CLI

推荐长期安装：

```powershell
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git
```

安装完成后验证：

```powershell
specify --help
specify check
```

如果你不想长期安装，也可以用一次性命令：

```powershell
uvx --from git+https://github.com/github/spec-kit.git specify --help
```

## 3. 分别安装三个工具

## 3.1 VS Code Copilot

### 安装步骤

1. 安装 VS Code

```powershell
winget install Microsoft.VisualStudioCode
```

2. 在扩展市场安装以下扩展：
   - `github.copilot`
   - `github.copilot-chat`
3. 使用你的 GitHub 账号登录 VS Code
4. 打开 Copilot Chat，确认能正常发消息

### 最简单的验证方式

打开 VS Code 后，看到右侧或顶部有 Copilot Chat 面板，能正常输入问题并得到回复，就说明已经可用。

## 3.2 Copilot CLI

Spec Kit 自己的开发容器脚本里，使用的安装命令是：

```powershell
npm install -g @github/copilot@latest
```

安装完成后先验证：

```powershell
copilot --help
```

如果首次启动要求登录，就按终端提示完成 GitHub 登录。

## 3.3 Claude Code

Spec Kit 自己的开发容器脚本里，使用的安装命令是：

```powershell
npm install -g @anthropic-ai/claude-code@latest
```

安装完成后先验证：

```powershell
claude --help
```

如果首次启动要求登录，就按终端提示完成认证。

## 4. 在项目里初始化 Spec Kit

下面以“当前目录已经是一个 Git 项目”为前提。

先进入你的项目目录：

```powershell
Set-Location D:\path\to\your-project
```

### 4.1 用 VS Code Copilot 或 Copilot CLI

这两个工具都用同一条初始化命令：

```powershell
specify init --here --ai copilot --script ps
```

初始化后，检查以下文件是否存在：

```powershell
Get-ChildItem .github\agents\speckit.code-review.agent.md
Get-ChildItem .github\prompts\speckit.code-review.prompt.md
Get-ChildItem .claude\skills
```

你看到这些文件，说明 Spec Kit 给 Copilot 的代码审查命令已经装好了。

但要注意：

1. `.github/prompts/speckit.code-review.prompt.md` 主要是给 VS Code Copilot 用的，它会让聊天框里出现可直接选择的 `/speckit.code-review`
2. Copilot CLI 实际读取的是 `.github/agents/speckit.code-review.agent.md`，不会像 VS Code 一样把 `/speckit.code-review` 做成同样的可选项

### 4.2 用 Claude Code

Claude 用单独的初始化命令：

```powershell
specify init --here --ai claude --script ps
```

初始化后，检查以下文件是否存在：

```powershell
Get-ChildItem .claude\commands\speckit.code-review.md
Get-ChildItem .claude\skills
```

你看到这些文件，说明 Spec Kit 给 Claude 的代码审查命令已经装好了。

## 5. 怎么用 Spec Kit 审查代码

下面给你三套完全照抄就行的步骤。

先记住一个最容易搞混的点：

`specify init`、`specify check` 这种命令是在普通 PowerShell 里执行。

`/speckit.code-review` 这种斜杠命令，不是在普通 PowerShell 里直接执行。

VS Code Copilot 和 Claude Code 可以直接输入 `/speckit.code-review`。

Copilot CLI 不走 `.github/prompts` 这一层，所以不是在 CLI 里直接输入 `/speckit.code-review`，而是通过 agent 方式调用 `speckit.code-review`。

## 5.1 在 VS Code Copilot 里审查代码

1. 用 VS Code 打开你的项目根目录
2. 打开 Copilot Chat
3. 直接输入下面这条命令

```text
/speckit.code-review review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

4. 如果它追问你开发者、时间范围、需求文档来源，就按提示回答
5. 等待它完成提交记录收集、调用链分析和报告生成
6. 去项目里的 `docs/review-results/` 目录查看报告

### 推荐回答模板

如果被问到是否有需求文档，你可以直接这样答：

```text
需求文档没有，按开发者提交记录做代码审查。
```

如果被问到时间范围，你可以这样答：

```text
开始日期 2026-04-13，结束日期 2026-04-13。
```

## 5.2 在 Copilot CLI 里审查代码

先说你现在看到的现象为什么会发生：

VS Code Copilot 会读取 `.github/prompts/speckit.code-review.prompt.md`，所以聊天框里会把 `/speckit.code-review` 当成一个可选命令展示出来。

Copilot CLI 不读取 `.github/prompts`，它读取的是 `.github/agents/speckit.code-review.agent.md`。所以在 Copilot CLI 里，你通常不会看到和 VS Code 一样的 `/speckit.code-review` 可选项。

Copilot CLI 的正确用法是：

1. 先在 PowerShell 里进入项目目录并启动 Copilot CLI
2. 在 Copilot CLI 里输入 `/agent`
3. 选择 `speckit.code-review`
4. 选中 agent 后，再输入真正的审查请求：`review 张三 zhangsan 在 op-api 2026-04-13 提交的代码`

如果你把 `/speckit.code-review ...` 直接敲在普通 PowerShell 里，或者直接敲在 Copilot CLI 里期待它像 VS Code 那样出现同名命令，通常都不会得到你想要的效果。

1. 打开 PowerShell
2. 进入项目目录

```powershell
Set-Location D:\path\to\your-project
```

3. 启动 Copilot CLI

```powershell
copilot
```

你看到 Copilot CLI 的交互提示符后，先输入：

```text
/agent
```

4. 在 agent 列表中选择 `speckit.code-review`

5. 选中后再输入：

```text
review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

6. 按提示回答补充问题
7. 审查完成后，到 `docs/review-results/` 查看本地报告

### Copilot CLI 最短照抄版

先在 PowerShell 执行：

```powershell
Set-Location D:\path\to\your-project
copilot
```

进入 Copilot CLI 后依次输入：

```text
/agent
```

选中 `speckit.code-review` 后，再输入：

```text
review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

如果你想一条命令直接跑，也可以这样调用：

```powershell
copilot --agent speckit.code-review "review 张三 zhangsan 在 op-api 2026-04-13 提交的代码"
```

### Copilot CLI 如果要走 Claude 兼容接口，配置要怎么改

这件事最容易搞混的地方是：Claude Code 的配置字段名，不能原样照搬到 Copilot CLI。

你的 Claude Code 配置里常见的是：

1. `ANTHROPIC_AUTH_TOKEN`
2. `ANTHROPIC_BASE_URL`
3. `ANTHROPIC_MODEL`
4. `ANTHROPIC_REASONING_MODEL`
5. `ANTHROPIC_DEFAULT_HAIKU_MODEL`
6. `ANTHROPIC_DEFAULT_OPUS_MODEL`
7. `ANTHROPIC_DEFAULT_SONNET_MODEL`

Copilot CLI 真正要认的是 Copilot 自己的 provider 环境变量：

1. `COPILOT_PROVIDER_TYPE`
2. `COPILOT_PROVIDER_BASE_URL`
3. `COPILOT_PROVIDER_API_KEY`
4. `COPILOT_MODEL`

也就是说：

1. `ANTHROPIC_*` 这套名字属于 Claude Code，不是 Copilot CLI 的正式 provider 配置名
2. Copilot CLI 更可靠的做法是把 provider 信息写到用户级环境变量，而不是指望 `config.json` 里的 `env` 节点生效
3. `ANTHROPIC_REASONING_MODEL` 和 `ANTHROPIC_DEFAULT_*` 这几项，在 Copilot CLI 里没有一一对应的配置项。Copilot CLI 一次只会选一个当前模型
4. 模型名必须写服务端真实可用的 model id，不要直接写营销名或 Claude Code 的别名。比如已经验证可用的是 `claude-opus-4-20250514`

PowerShell 可以这样设置用户级环境变量：

```powershell
[Environment]::SetEnvironmentVariable("COPILOT_PROVIDER_TYPE", "anthropic", "User")
[Environment]::SetEnvironmentVariable("COPILOT_PROVIDER_BASE_URL", "https://cc.580ai.net", "User")
[Environment]::SetEnvironmentVariable("COPILOT_PROVIDER_API_KEY", "<your-api-key>", "User")
[Environment]::SetEnvironmentVariable("COPILOT_MODEL", "claude-opus-4-20250514", "User")
```

设置完成后，关闭当前终端，再重新打开 Copilot CLI。

如果你还想把 Claude Code 里的 hooks 迁过来，也不要继续用 `checkpoint claude`，而要改成 Copilot 对应的 checkpoint 名称：`checkpoint github-copilot --hook-input stdin`。

最稳妥的 Copilot hooks 形状可以写成：

```json
{
   "hooks": {
      "PreToolUse": [
         {
            "type": "command",
            "command": "C:\\Users\\admin\\.git-ai\\bin\\git-ai.exe checkpoint github-copilot --hook-input stdin"
         }
      ],
      "PostToolUse": [
         {
            "type": "command",
            "command": "C:\\Users\\admin\\.git-ai\\bin\\git-ai.exe checkpoint github-copilot --hook-input stdin"
         }
      ]
   }
}
```

这段内容可以放在 Copilot 的用户级 hook 配置里，也可以按仓库维度放到 `.github/hooks/` 下的 JSON 文件中。

## 5.3 在 Claude Code 里审查代码

这套流程和 Copilot CLI 一样，也是两层：

1. 先在 PowerShell 里进入项目目录并启动 Claude Code
2. 再在 Claude Code 的交互界面里输入 `/speckit.code-review`

同样不要把 `/speckit.code-review ...` 直接敲在普通 PowerShell 里。

1. 打开 PowerShell
2. 进入项目目录

```powershell
Set-Location D:\path\to\your-project
```

3. 启动 Claude Code

```powershell
claude
```

Claude Code 读取的是 `.claude/commands/speckit.code-review.md`，所以这里就是直接输入斜杠命令，不需要像 Copilot CLI 那样先切 agent。

4. 在 Claude Code 交互界面中输入：

```text
/speckit.code-review review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

5. 按提示回答补充问题
6. 审查完成后，到 `docs/review-results/` 查看本地报告

### Claude Code 最短照抄版

先在 PowerShell 执行：

```powershell
Set-Location D:\path\to\your-project
claude
```

进入 Claude Code 后再输入：

```text
/speckit.code-review review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

### Claude Code 额外注意

如果 `claude` 能启动，但在执行过程中提示认证、权限或联网问题，先完成 Claude Code 登录，再重新进入项目目录执行同样的步骤。

如果你已经执行过 `specify init --here --ai claude --script ps`，但在 Claude Code 里没有看到命令提示，先确认你是从当前项目根目录启动的 `claude`，然后重启 Claude Code 再试。即使没有下拉提示，也可以直接手动输入 `/speckit.code-review ...`。

## 6. 常用输入模板

## 6.1 按开发者审查

```text
/speckit.code-review review 张三 zhangsan 在 op-api 2026-04-13 提交的代码
```

适用场景：你明确知道是谁、哪个仓库、哪一天的提交。

## 6.2 按时间范围审查

```text
/speckit.code-review review 张三 zhangsan 在 op-api 2026-04-10 到 2026-04-13 提交的代码
```

适用场景：要看一个开发周期内的多次提交。

## 6.3 按需求相关代码审查

```text
/speckit.code-review review 订单批量取消需求相关代码，时间范围 2026-04-10 到 2026-04-13
```

适用场景：你手里有需求背景，但不想只按某个开发者过滤。

## 6.4 审查当前未提交修改

```text
/speckit.code-review review 当前工作区未提交代码
```

适用场景：代码还没 push 或还没 commit，先做一轮本地自查。

## 7. 你应该看到什么结果

正常情况下，Spec Kit 的代码审查流程会依次做这些事：

1. 识别仓库和开发者信息
2. 收集提交记录或本地 diff
3. 调用子代理分析调用链
4. 套用前后端审查规则
5. 生成审查报告

本地报告默认看这个目录：

```text
docs/review-results/
```

典型文件名类似：

```text
docs/review-results/code-review-YYYY-MM-DD-作者名.md
```

如果环境里配置了额外的 MCP 服务器，它还可能继续同步远程问题单；如果没有，也不影响本地报告生成。

## 8. 最常见的故障和处理方法

## 8.1 `/speckit.code-review` 不识别

先检查命令文件有没有生成。

Copilot：

```powershell
Get-ChildItem .github\agents\speckit.code-review.agent.md
Get-ChildItem .github\prompts\speckit.code-review.prompt.md
```

Claude：

```powershell
Get-ChildItem .claude\commands\speckit.code-review.md
```

如果文件存在但命令不识别：

1. 彻底重启 VS Code、Copilot CLI 或 Claude Code
2. 确认你当前打开的就是执行过 `specify init` 的目录
3. 再执行一次初始化命令

## 8.2 `specify init` 失败

先跑诊断：

```powershell
specify check
```

如果是当前目录非空导致的确认问题，可以强制执行：

```powershell
specify init --here --force --ai copilot --script ps
specify init --here --force --ai claude --script ps
```

## 8.3 审查结果为空

优先检查这三件事：

1. 当前目录是不是 Git 仓库
2. 开发者名字、邮箱、日期是不是写对了
3. 这段时间里是不是真的有提交记录

你可以先自己验证一遍 Git 日志：

```powershell
git log --oneline --decorate --since="2026-04-13 00:00:00" --until="2026-04-13 23:59:59"
```

## 8.4 Copilot 和 Claude 该怎么选

如果你只想要一个最简单的判断规则，直接照这个用：

1. 你主要在 VS Code Chat 里工作，用 VS Code Copilot
2. 你主要在终端里工作，但想继续用 GitHub 生态，用 Copilot CLI
3. 你希望整个流程都在 CLI 里，且命令入口更直接，用 Claude Code
